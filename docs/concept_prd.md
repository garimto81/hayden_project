# 🚀 [PRD] 키워드 레이더 (Keyword Radar)

**Version:** 1.0.0
**Date:** 2025-05-20 (Target)
**Status:** Draft (기획 확정 단계)

## 1\. 프로젝트 개요 (Overview)

  * **제품명:** 키워드 레이더 (Keyword Radar)
  * **슬로건:** "지금 안 보면 떡락\! 마케터를 위한 실시간 트렌드 호가창"
  * **핵심 목표:** 정적인 트렌드 데이터를 **주식/코인 거래소의 긴박한 UI**로 시각화하여, 마케터에게 정보 습득의 재미와 속도감을 동시에 제공.
  * **타겟 유저:** 트렌드에 민감해야 하는 콘텐츠 마케터, 에디터, 바이럴 대행사 직원.

-----

## 2\. 유저 스토리 (User Story)

1.  **진입 시점 (The Hook):** 유저는 페이지에 접속하자마자 정적인 텍스트가 아닌, 숫자가 `0%`에서 `+1,200%`까지 빠르게 굴러가는(Rolling) 애니메이션을 보며 "지금 뭔가 터지고 있다"는 긴박함을 느낀다.
2.  **탐색 시점 (Discovery):** 실시간으로 순위가 뒤바뀌는 리스트를 보며, 빨간 불기둥(급상승)이 뜬 키워드를 본능적으로 클릭한다.
3.  **소비 시점 (Consumption):** 상세 페이지에서 주가 차트 형태의 그래프를 확인하고, '종토방'에서 다른 마케터들의 드립을 보며 소속감을 느낀다.

-----

## 3\. 상세 기능 명세 (Functional Specs)

### 3.1. 프론트엔드 (Frontend) - "Visual Dopamine"

**기술 스택 제안:** Next.js 14+ (App Router), Tailwind CSS, Framer Motion (애니메이션)

| 구분 | 상세 기능 | 기술/UI 요구사항 (개발자 전달용) |
| :--- | :--- | :--- |
| **메인 대시보드** | **실시간 롤링 티커** | - 페이지 로드 시 `CountUp` 라이브러리 사용 (숫자 롤링 효과).<br>- 데이터 갱신 시 숫자가 깜빡이며 변경 (Flash Effect). |
| | **호가창 리스트 UI** | - 상승률에 따라 배경색 채도 변경 (예: +10% 옅은 빨강 → +100% 진한 네온 레드).<br>- 코인 거래소와 동일한 레이아웃 (좌측: 순위/키워드, 우측: 등락률/화력). |
| | **VI (변동성 완화) 연출** | - 5분 내 30% 이상 급등 시: 해당 Row가 붉게 `Pulse` 애니메이션.<br>- 키워드 옆 `[🚨VI발동]` 배지 부착. |
| **상세 페이지** | **캔들 차트** | - **TradingView Lightweight Charts** 라이브러리 사용 권장 (가볍고 주식 차트 느낌 완벽 구현).<br>- X축: 시간(5분 단위), Y축: 언급량/조회수 합산. |
| **커뮤니티** | **실시간 종토방** | - 로그인 없는 익명 채팅 (닉네임 자동 생성: '물린 마케터 1').<br>- [떡상/떡락] 투표 버튼 클릭 시 즉시 % 게이지 변화 애니메이션. |

### 3.2. 백엔드 & 데이터 (Backend) - "The Engine"

**기술 스택 제안:** Python (FastAPI), Supabase (DB & Realtime), Selenium/Playwright

| 구분 | 상세 기능 | 로직/알고리즘 요구사항 |
| :--- | :--- | :--- |
| **수집기 (Crawler)** | **멀티 스레드 크롤링** | - 대상: 펨코(포텐), 더쿠(핫게), 인스티즈, 블라인드(실시간), 네이트판.<br>- 주기: **5분** (서버 부하 고려하여 사이트별 텀 조정).<br>- **핵심:** `User-Agent` 로테이션으로 차단 방지. |
| **AI 전처리** | **키워드 추출 및 정제** | - **Kiwi(한국어 형태소 분석기)** 사용 (속도 최우선).<br>- 불용어 처리: '오늘', '진짜', 'ㅋㅋ', '속보' 등 무의미한 단어 제외.<br>- **유사어 통합(Embedding):** `sentence-transformers` 모델을 사용해 '지디'='GD'='권지용'을 같은 클러스터로 묶음 (형이 구현 필요). |
| **랭킹 엔진** | **화력(Momentum) 산출** | - **공식:** `(현재 구간 언급량 - 전 구간 언급량) / 전 구간 언급량 * 100`<br>- 가중치: 조회수 1점, 댓글수 5점 (댓글이 많은 것이 진짜 핫한 이슈). |

### 3.3. 수익화 (BM) - 네이티브 광고

  * **기능:** 랭킹 3\~5위 사이에 광고 슬롯 배치.
  * **연출:** 광고임에도 불구하고 `+154% 🔥` 같은 수치를 임의로 부여하여 시선 강탈.
  * **라벨:** 아주 작게 `AD` 표기 (법적 문제 회피용).

## 4\. 기술 아키텍처 및 데이터 흐름 (Architecture)

1.  **Collect:** Python Worker가 5분마다 커뮤니티 순회 → Raw Data 수집.
2.  **Process:** 형태소 분석 & 유사어 그룹핑 → '화력 지수' 계산.
3.  **Store:** Supabase DB에 `(키워드, 시간, 화력, 원문링크)` 저장.
4.  **Serve:**
      * Next.js 프론트엔드가 Supabase를 통해 데이터 Fetch.
      * **Realtime:** Supabase Realtime 기능을 켜서, DB에 새 데이터가 꽂히자마자 프론트엔드 숫자가 자동으로 갱신되도록 설정 (새로고침 없이 숫자가 바뀌어야 '도파민'이 터짐 ⭐).
